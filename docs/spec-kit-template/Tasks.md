# Tasks 양식 (Spec Kit Template)

> Copilot Workspace가 Tasks를 생성할 때 이 양식을 참조합니다.

## Tasks

- [ ] **Task 1**: `app/models/xxx.py` — 구체적인 작업 내용 (예: 새 필드 추가)
- [ ] **Task 2**: `app/data/xxx.py` — 구체적인 작업 내용 (예: 저장소 로직 반영)
- [ ] **Task 3**: `app/services/xxx.py` — 구체적인 작업 내용 (예: 비즈니스 로직 업데이트)
- [ ] **Task 4**: `app/routers/xxx.py` — 구체적인 작업 내용 (예: 라우트 수정, 필요 시)
- [ ] **Task 5**: `frontend/src/types.ts` — 타입 정의 동기화
- [ ] **Task 6**: `frontend/src/components/xxx.tsx` — UI 컴포넌트 업데이트
- [ ] **Task 7**: 검증 — 백엔드 API + 프론트엔드 UI 변경 사항 검증

### 주의사항
- 하나의 Task = 하나의 파일 변경 (가능한 한)
- 각 Task는 독립적으로 검증 가능
- 마지막 Task는 반드시 검증/테스트
